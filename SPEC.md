# MusicPlayer - macOS 音乐播放器规格说明书

## 1. 项目概述

- **项目名称**：MusicPlayer
- **Bundle Identifier**：com.dashu.musicplayer
- **核心功能**：极简 macOS 音乐播放器，仅限播放控制 + 歌单同步
- **目标用户**：大树（macOS 用户）
- **最低 macOS 版本**：macOS 13.0+

---

## 2. UI/UX 规格

- **主窗口**（单窗口）：800×600，最小 600×400，支持缩放
- **无额外弹窗**：所有操作在主窗口内完成

## 3. 功能规格

### 核心功能

**F1. 歌单同步**
- 支持平台：QQ音乐、网易云音乐
- 同步方式：
  - QQ音乐：解析公开歌单分享链接（https://y.qq.com/n/ryqq/playlist/xxx）
  - 网易云音乐：解析公开歌单分享链接（https://music.163.com/playlist?id=xxx）
- 登录方式：平台账号密码登录（Web 授权或 API）
- 同步内容：歌单名称、歌曲列表（标题、艺术家、专辑、时长）
- 同步频率：手动刷新 + 自动检测更新
- 存储：本地 SQLite，歌曲数据缓存

**F2. 音乐播放**
- 播放控制：播放、暂停、上一首、下一首
- 进度控制：进度条显示 + 拖拽跳转
- 音量控制：滑块调节（0-100%）
- 播放模式：顺序播放、随机播放、循环播放（单曲/列表）
- 歌曲搜索：在当前歌单内搜索
- 列表管理：显示当前歌单歌曲，双击播放

**F3. 歌词显示（可选，当前版本不含）**
- v1.0 不含歌词功能

### 用户交互流程

1. 启动 → 显示空状态（引导绑定平台账号）
2. 点击"添加平台" → 弹出平台选择 → 输入账号密码
3. 绑定成功 → 显示歌单列表
4. 点击歌单 → 右侧显示歌曲列表
5. 点击歌曲 → 开始播放，底部控制栏激活
6. 点击"同步" → 刷新当前歌单歌曲列表

### 数据模型

```
Playlist
  - id: String
  - platform: Platform (QQ | NetEase)
  - name: String
  - songCount: Int
  - lastSyncTime: Date?
  - songs: [Song]

Song
  - id: String
  - title: String
  - artist: String
  - album: String?
  - duration: Int (秒)
  - playUrl: String? (缓存的音频 URL)
  - platform: Platform

Platform
  - QQ: 腾讯QQ音乐
  - NetEase: 网易云音乐
```

### 错误处理

- 网络错误：弹 Toast 提示"网络连接失败"
- 同步失败：显示具体失败原因（无效链接/账号异常）
- 播放失败：自动跳到下一首，提示"播放失败"
- 平台未登录：显示登录引导

## 4. 技术规格

- **语言**：Swift 5.9+
- **UI 框架**：SwiftUI（macOS 13+）
- **数据存储**：SQLite.swift
- **网络**：URLSession + async/await
- **音频**：AVFoundation (AVPlayer)
- **架构**：MVVM
- **打包**：XcodeGen + DMG
- **图标**：SF Symbols（系统自带，无自定义图标）

### 第三方依赖（Swift Package Manager）

```
SQLite.swift        - 本地数据存储
```

## 5. Google UI 参考要点

**参考产品**：Google Photos（macOS Web）、Google Gemini App
**核心特征**：
- 大量留白，信息密度低
- 圆角卡片承载内容
- 图标用 SF Symbols，线条简洁
- 主色点缀，强调操作入口
- 无阴影层次感，靠背景色区分层级
- 过渡动画：200ms ease-out
